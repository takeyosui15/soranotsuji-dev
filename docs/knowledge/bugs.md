# バグ記録（ノウハウ集）

バグの傾向と教訓を蓄積するドキュメントです。
同じ種類のバグを繰り返さないために、「何が起きて、なぜ起きて、何を学んだか」を記録します。

> バグトラッカーではありません。今直すべきバグは `todo.md` に書きます。
> ここは「直した後の学び」を残す場所です。

## 記録の書き方

```markdown
## バグ: タイトル（日付）
### 現象
- 何が起きたか
### 原因
- なぜ起きたか
### 教訓
- 次に同じことを防ぐために何を覚えておくか
```

### 現象
> 何が起きたか.

### 原因
> なぜ起きたか.
