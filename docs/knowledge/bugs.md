バグの傾向と教訓を蓄積するドキュメントです。
同じ種類のバグを繰り返さないために、「何が起きて、なぜ起きて、何を学んだか」を記録します。

> バグトラッカーではありません。今直すべきバグは `todo.md` に書きます。
> ここは「直した後の学び」を残す場所です。

## 記録の書き方

```markdown

- 何が起きたか
### 原因
- なぜ起きたか
### 教訓
- 次に同じことを防ぐために何を覚えておくか
```

## バグ: ヘルプの更新漏れ(2026-03-11)
### 現象
辻検索メニューのテキストボックスの並び順を変更したが、ヘルプの説明文が古い並び順のままだった。検索期間の説明もヘルプに記載されていなかった。

### 原因
UI変更時にヘルプの更新を忘れていた。ヘルプはユーザーが直接見る部分だが、開発中は動作確認に集中してしまい、ヘルプの内容確認が漏れやすい。

### 教訓
UI変更時のコミット前チェックリスト:
- [ ] ヘルプの記載内容が実際のUIと一致しているか
- [ ] 追加・変更したテキストボックスのプレースホルダーが正しいか
- [ ] 全角/半角の表記ゆれがないか
- [ ] 並び順がUIと一致しているか
