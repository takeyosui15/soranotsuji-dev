My辻検索、My観測点、My目的点の「全て登録」ボタンは、存在に意味はあるのでしょうか。
なんだか、逐次リアルタイムにデータがappStateとローカルストレージで同期が取れているような気がするのですが…。

### 回答 (2026-04-22) — 「全て登録」ボタンの役割について

ご指摘の通り、**データの保存自体はリアルタイムで行われています**。各フィールドの `change` イベントで `saveAppState()` が呼ばれるため、入力内容は即座に appState + localStorage に反映されます。

ただし、「全て登録」ボタンにはリアルタイム保存にない以下の役割があります:

| 役割 | 内容 |
|---|---|
| **バリデーション** | 未入力の必須項目 (名前、緯度経度など) をチェックし、エラーメッセージを表示 |
| **半角カンマ→全角変換** | 名前・メモ内の半角カンマ `,` を全角 `，` に一括変換 (CSV崩れ防止) |
| **dirty flag クリア** | 「全て登録」ボタンの赤色太字表示を通常に戻す |
| **マーカー更新** | My観測点/My目的点の地図マーカーを再描画 (My辻検索にはなし) |
| **ユーザーへの安心感** | 確認ダイアログ + 完了メッセージで「保存した」実感を提供 |

**結論**: データ永続化の観点では不要ですが、**バリデーション**と**カンマ変換**のために残しておく価値があります。特にカンマ変換は CSV 出力時の列崩れ防止に必要で、リアルタイム保存では行っていません。

もし不要と判断される場合は、バリデーションとカンマ変換のロジックをリアルタイム保存側に移設して「全て登録」ボタンを廃止することも可能です。
