#### 2. work ブランチの作成方法

試行環境のローカルフォルダ (soranotsuji-sandbox-local) で以下を実行してください:

```bash
cd "$HOME/Library/Mobile Documents/com~apple~CloudDocs/Documents/soranotsuji-sandbox-local"

# 1. main の最新を取得
git checkout main
git pull origin main

# 2. develop ブランチが無い場合は作成
git checkout -b develop
git push -u origin develop

# 3. develop から work ブランチを作成
git checkout -b work
git push -u origin work

# 4. ブランチ一覧を確認
git branch -a
# * work
#   develop
#   main
#   remotes/origin/develop
#   remotes/origin/main
#   remotes/origin/work
```

これで試行環境でも開発環境と同じ `claude/xxx → work → develop → main` のフローで作業できます。

work ブランチでの日常作業:
```bash
# work ブランチで作業
git checkout work

# 作業・コミット
git add <files>
git commit -m "内容"
git push origin work

# 固まったら develop にマージ
git checkout develop
git merge work
git push origin develop
```

### 回答 (2026-04-22) — 南中時/視半径の表示修正 + 辻検索取得の精度フィルタ反映

#### 1. 南中時行に視半径が表示されない問題の修正
**原因**: 南中時の設定は `setTimeout` で非同期実行されるため、先に同期実行される方位角/視高度セッターが `transit-{id}` に「南中時 + 視半径」を書き込んでも、後から南中時セッターが「南中時 XX:XX:XX」だけで**上書き**していた。

**修正**:
- 恒星・太陽系天体両方の `setTimeout` 内の南中時セッターで、既存の `/ 視半径 ...` 部分を保持するよう変更
- 初期表示テキストにも `/ 視半径` を含めるよう変更 (恒星: `-.---°`、太陽系: `--°`)

#### 2. 辻検索取得で精度フィルタが反映されない問題の修正
**原因**: `getMyTsujiFromTsujiSearch()` で `moonFilter` は `appState.tsujiMoonFilterEnabled` からコピーしていたが、精度フィルタ (`accuracyFilter`, `accDblCircle`, `accCircle`, `accTriangle`, `accDash`) のコピーが漏れていた。

**修正**: `getMyTsujiFromTsujiSearch()` に精度フィルタ5項目のコピーを追加。`memo: ''` も欠落していたので併せて追加。
