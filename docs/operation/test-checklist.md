作成日: 2026-03-17

デプロイ前に確認すべきテスト項目をまとめたチェックリスト。
developブランチでGitHub Pagesに反映後、このリストに従ってテストを行う。

## テスト方法の段階

### Phase 1: 手動テスト（現在）

このチェックリストに従い、ブラウザで手動確認する。
テスト環境: `https://takeyosui15.github.io/soranotsuji-dev/`

### Phase 2: ブラウザ開発者ツールの活用

Chrome DevTools を使った確認:
- **Console タブ**: JavaScriptエラーの確認（赤いエラーが出ていないか）
- **Network タブ**: API呼び出し（標高データ、地図タイル）が正常か
- **Application タブ**: LocalStorageの保存・読込が正常か
- **Performance タブ**: 描画速度やメモリ使用量の確認

### Phase 3: 自動テスト（将来検討）

デプロイ頻度や機能の複雑さが増した場合に検討:
- **Playwright**: ブラウザを自動操作してUIテスト
  - 「テキストボックスに値を入力→ボタンクリック→結果を確認」を自動化
  - GitHub Actionsと連携して、pushのたびに自動テスト
- ただし静的サイト（ビルドなし）では導入コストが高いため、手動テストで十分な段階が長い

## デプロイ前テストチェックリスト

### 基本動作

- [ ] ページが正常に読み込まれる（白画面やエラーにならない）
- [ ] Chrome DevTools の Console にエラー（赤文字）が出ていない
- [ ] 地図が正常に表示される（Leaflet）
- [ ] 地図タイルが読み込まれる（国土地理院タイル）

### 天体計算

- [ ] 天体の方位角・高度が計算される
- [ ] 日時を変更すると天体位置が更新される
- [ ] 太陽・月・惑星・恒星が正しく選択・表示できる

### 辻検索

- [ ] 辻検索ボタンが動作する
- [ ] 検索結果がリスト表示される
- [ ] 検索結果をクリックすると地図上に表示される

### データ保存・復元

- [ ] LocalStorageへの保存が正常に動作する
- [ ] ページリロード後にデータが復元される
- [ ] 設定（観測点、天体選択など）が保持される

### 入力バリデーション

- [ ] テキストボックスに異常な値（文字列、極端な数値）を入力してもエラーにならない
- [ ] 入力範囲（min/max）が正しく機能する
- [ ] step属性が正しく機能する

### レスポンシブ・表示

- [ ] PC（Chrome）で正常表示
- [ ] スマートフォン（Safari/Chrome）で正常表示
- [ ] ヘルプが正常に表示される
- [ ] MathJax（数式）が正常にレンダリングされる

## Chrome DevTools の開き方

1. テスト対象のページを Chrome で開く
2. 以下のいずれかで DevTools を開く:
   - キーボード: **F12** または **Cmd + Option + I**（Mac）
   - メニュー: **表示** → **開発/管理** → **デベロッパー ツール**
   - ページ上で右クリック → **検証**
3. **Console** タブを確認（赤いエラーがなければOK）
