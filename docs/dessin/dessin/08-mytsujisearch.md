# デッサン（dessin）

## My観測点メニュー
-------------------------------------------------------------------------------
-------------------------------------------------------------------------------


### 目的:
- My観測点メニューとMy目的点メニューとMy天体メニューと併せて用いることで、My辻検索を実現し、辻検索を複数登録することができて、一括で計算、ファイル出力できるものとする。
### 概要:
- 機能概要
  - 辻検索を複数登録でき、My辻検索リストとして管理できる。
  - My辻検索の追加・削除ができる。
  - My辻検索のリストをCSVファイルで入力・出力できる。
  - 辻検索メニューの辻検索情報をMy辻検索に簡単に追加できる。
  - 選択中のMy辻検索を一括で簡単に計算できる。
- デザイン概要
  - 薄明メニューの下に配置する。
  - 0段目:
    - 1. 「My辻検索」ラベル:
      - 太字、左寄せ
    - 2. 「▼」ラベル:
      - 右寄せ
    - 共通事項:
      - タップすると、My辻検索メニューが開閉する。
      - 背景グレー、不透明
      - 初期状態:閉じている。
  - 1段目:
    - 3. 「一括計算」ボタン:
      - このボタンを押下すると、6.ラジオボタンで選択されている観測点の緯度・経度・高さを位置情報メニューの観測点情報に設定して、画面更新をする。
      - このボタンを押下すると、実行する前に、確認メッセージ「観測点（ID:(数字)、観測点名）を位置情報メニューと地図に反映しますか？」（window.confirmで、OK、キャンセル）が表示される。
        - 地図の画面の中心に選択中のMy観測点が表示される。
    - 4. 「一括File取得」
  - 2段目:
    - 5. 「辻検索取得」ボタン:
      - このボタンを押下すると、位置情報メニューの観測点情報に設定されている観測点の緯度・経度・高さを1件の新規My観測点として、新規にIDを採番して、My観測点リストに末尾に行を追加して追加する。
        - 新規My観測点の初期値
          - 7. 観測点ID:14.行追加ボタンと同様の採番方法
          - 8. 観測点名:「新規観測点名」
          - 9. 観測点緯度経度:位置情報メニューの観測点の緯度経度
          - 10. 観測点標高:位置情報メニューの観測点の標高
          - 11. 観測点高:位置情報メニューの観測点の高さ
      - このボタンを押下すると、実行する前に、確認メッセージ「現在の位置情報（緯度・経度・高さ）をMy観測点リストに追加しますか？」（window.confirmで、OK、キャンセル）が表示される。
    - 6. 「全て登録」ボタン:
      - このボタンを押下すると、ローカルストレージのMy観測点リストに、各観測点情報（観測点ID、観測点名、観測点緯度経度、観測点標高、観測点高）を保存する。動作としては、ローカルストレージのMy観測点リストを一旦全て削除してから、全件追加する。
      - 登録前のチェック内容として、1件のMy観測点情報に未入力の内容があった場合は、登録できず、以下のメッセージを赤字太字で表示し、その後、未入力のフォームにフォーカスして、フォームが赤枠で表示されて、メニューがスクロールされる。
        - 「観測点ID:(数字)に未入力のものがあります。入力するか、行削除してください。」
      - 登録前のチェックが終わった後に、確認メッセージ「現在のMy観測点リストをローカルストレージに登録しますか？」（window.confirmで、OK、キャンセル）が表示される。
        - 登録時には、観測点名テキストボックス以外のテキストボックスの内容は、全て全角から半角に変換してから登録する。
      - My観測点リストのフォームに変更があった場合（観測点取得、行追加、行削除、表示位置の上下移動、フォーム内容の変更）は、ボタンのデザインが赤色太字に変更される。
      - My観測点が全て登録されると、地図上に、My観測点1件につき、1つのMy観測点マーカーが表示される。
        - My観測点マーカーの詳細
          - マーカー色:緑
          - ポップアップの内容:
            - タイトル:「My観測点」(太字)
            - 観測点名:「(観測点名)」
            - 緯度:「緯度: 35.65858°」
            - 経度:「経度: 139.74547°」
            - 標高:「標高: 18.5 m」
            - 高さ:「高さ: 150.0 m」
          - ポップアップをクリックすると、そのMy観測点を位置情報の観測点に設定し、観測点マーカー(青)をそこに移動させる。
          - ポップアップを長押しすると、確認メッセージを表示して、クリップボードにMy観測点のURLをコピーする。
            - 確認メッセージ「現在のMy観測点をクリップボードにコピーしますか？」（window.confirmで、OK、キャンセル）が表示される。
  - 3〜N段目:My観測点の数だけ繰り返す
    - 凡例:0️⃣:段抜き、1️⃣:1行目、2️⃣:2行目、以下同様
    - 0️⃣:7. ラジオボタン:
      - このボタンで選択されている観測点は、3.「位置反映」ボタン、4.「観測点取得」ボタン、10.「行削除」ボタン、の操作対象であることを示す。
    - 0️⃣:8. チェックボックス:
    - 0️⃣:9. 辻検索ID「(数字)」ラベル:
      - 制約:辻検索を一意に定める数字。My辻検索のキー。
      - 制約:最小値1から始まって、最大値1000まで。
    - 1️⃣:10. 辻検索名テキストボックス:
      - 辻検索に対して、分かりやすい名前を入力できるようにする。
      - 名前は重複しても良い。
      - プレースホルダー:「辻検索名」
    - 2️⃣:11. 検索期間テキストボックス:
      - 検索期間を日で指定する。
      - 初期値:365
      - 制約:1〜1461(365+365+365+366、4年間)、ステップ1
      - プレースホルダー:「検索期間(日)」
    - 3️⃣:10. 観測点標高テキストボックス:
      - 観測点の緯度/経度から標高を表示する。
      - 標高は、9.観測点緯度経度が更新された時に再計算される。
      - 背景:灰色
      - 制約:読み取り専用
      - 書式:標高を数字で表示する。
      - 例:18.9
    - 4️⃣:11. 観測点高テキストボックス:
      - 観測点の追加の高さを設定する。
      - 背景:灰色
      - 書式:高さを数字で入力する。
      - 例:150
  - N+1段目
    - 12. 「▲上へ」ボタン:
      - ラジオボタンで選択中のMy観測点を1行上へリスト内の表示の位置を移動する。
    - 13. 「▼下へ」ボタン:
      - ラジオボタンで選択中のMy観測点を1行下へリスト内の表示の位置を移動する。
  - N+2段目
    - 14. 「行追加」ボタン:
      - このボタンを押下すると、2段目〜N段目の中で、ラジオボタンで選択中の行の次の行へ、1件の観測点情報（ラジオボタン、観測点ID、観測点名、観測点緯度経度、観測点標高、観測点高）を追加する。動作としては、IDを採番して、appState、ローカルストレージに空の観測点情報を追加する。
        - 追加My観測点の初期値
          - 7. 観測点ID:下記採番方法
          - 8. 観測点名:空白
          - 9. 観測点緯度経度:空白
          - 10. 観測点標高:空白
          - 11. 観測点高:0
        - 7.観測点IDの採番は、ラジオボタンで選択中の行のIDから連続して連番になる最小の番号を採番する。
          - 現在1,2,4,5が発番されている場合で、ラジオボタンでID:2が選択されている場合は、次の採番は、3になる。
      - 制約:7.観測点IDの制約の最大値を超えて追加しようとすると、このボタンが非活性（押せない状態）になる。
      - このボタンを押下すると、実行する前に、確認メッセージ「観測点リストの末尾に観測点の行を追加しますか？」（window.confirmで、OK、キャンセル）が表示される。
    - 15. 「行削除」ボタン:
      - このボタンを押下すると、ラジオボタンで選択された、1件の観測点情報（ラジオボタン、観測点ID、観測点名、観測点緯度経度、観測点標高、観測点高）を削除する。動作としては、appState、ローカルストレージに観測点IDの観測点情報を削除する。
      - 制約:7.観測点IDの制約の最小値を超えたものを削除しようとすると、このボタンが非活性（押せない状態）になる。
        - つまり、0件のMy観測点の時は、ボタンが非活性になる。
      - このボタンを押下すると、実行する前に、確認メッセージ「観測点リストの観測点（ID:(数字)、観測点名）を削除しますか？」（window.confirmで、OK、キャンセル）が表示される。
  - N+3段目
    - 17. 「CSV入力」ボタン:
      - CSVファイルからMy観測点のリストを入力できる。
        - 5列目:標高が空要素「,,」である場合、標高を取得して再計算してから登録する。
      - このボタンを押下すると、実行する前に、確認メッセージ「CSVファイルから全て上書き入力・登録しますか？」（window.confirmで、OK、キャンセル）が表示される。
        - 確認メッセージの後にファイル選択ダイアログが開いて、デバイス内のファイルを選択できる。
      - CSVファイルから入力されると現在のリスト全て削除して、CSVファイルの内容で全て上書きをして、appState、ローカルストレージにも登録される。
      - CSVの見出し行は、1列目:ID,2列目:観測点名,3列目:緯度,4列目:経度,5列目:標高,6列目:高さとして、見出し行が存在する。
      - 制約:IDの数字の小さいものから順に登録する。
      - 制約:IDの数字の重複チェックをする。
      - 制約:CSVで登録できる件数の上限は1000件まで。CSVの見出し行を入れて1001行まで。
    - 18. 「CSV出力」ボタン:
      - My観測点のリストをCSV出力できる。
      - このボタンを押下すると、実行する前に、確認メッセージ「CSVファイルに出力しますか？」（window.confirmで、OK、キャンセル）が表示される。
        - 確認メッセージの後にブラウザからCSVファイルをダウンロードできる。
  - N+4段目
    - 19. 「URL取得」ボタン:
      - 宙の辻のURLにクエリストリングを渡して、開くURLを取得する。
        - ラジオボタンで選択中のMy観測点を現在の観測点とするURLを取得する。
        - 観測点情報以外は、全て既定の値を用いて(つまりキーを指定しないで)URLを作成する。
        - URLをクリップボードにコピーすると、次の確認メッセージ「観測点リストの観測点（ID:(数字)、観測点名）を開くURLをクリップボードにコピーしました。」（window.alertで、OK）が表示される。
- 共通機能/デザイン
  - テキストボックス:
    - 未入力でフォーカスが外れた場合、元の値を復元する。
    - title属性でテキストボックスの内容をツールチップで表示する。
    - input:type=textは、最大文字数150文字とする。
    - XXS対策で、入力内容をタグエスケープする。
    - フォーム、appState、ローカルストレージで、整合性が取れるようにする。
  - 背景色:
    - 1〜N+1段目は、背景白で、少し透明で、背景の地図が見える。
- コントロールの初期状態
  - ローカルストレージのMy観測点リストの登録内容によって、登録されている内容を各My観測点ごとに、2段目以降に行を追加して、コントロールに読み込んで表示する。
    - 読み込みに時間がかかる場合もあるので、非同期処理として実装する。
  - ローカルストレージのMy観測点リストがない場合は、0件（2〜N段目がない状態）になる
- 地図上のマーカーの初期状態
  - 各My観測点にMy観測点マーカー(緑)で地図上に位置を表示する。
    - 読み込みに時間がかかる場合もあるので、非同期処理として実装する。
