# デッサン（dessin）
-------------------------------------------------------------------------------
-------------------------------------------------------------------------------
## 位置情報メニュー
-------------------------------------------------------------------------------
-------------------------------------------------------------------------------


### 目的:
- ダイヤモンド富士が見える場所をわかりやすく伝えるために、辻ラインを表示するが、それを計算すための基本情報の観測点/目的点の緯度/経度/標高を入力できるようにする。
### 概要:
- 0段目:
  - 1. 「位置情報」ラベル:
    - 太字、左寄せ
  - 2. 「▲」ラベル:
    - 右寄せ
  - 共通事項:
    - タップすると、位置情報メニューが開閉する。
    - 背景グレー、不透明
    - 初期状態:開いている
- 1列目:
  - 1段目:
    - 3. 「GPS」ボタン:
      - 現在地を取得すると、ラジオボタンで選択された方の観測点/目的点に緯度/経度/標高が入力される。
      - スマホで現在地を取得できるようにする。
  - 2段目:
    - 4. 「Hom(e)」ボタン:ラベル「Hom」
      - 観測点を登録しておくためのもの。観測点が空白で、リセットできる。
      - ローカルストレージに保存して、観測点情報(緯度経度標高)を呼び出せるようにする。
      - 押したらどうなるか分かるメッセージをtitle属性でツールチップで表示する。
  - 3段目:
    - 5. 「推(し)山」ボタン:ラベル「推山」
      - 目的点を登録しておくためのもの。目的点が空白で、リセットできる。
      - ローカルストレージに保存して、目的点情報(緯度経度標高)を呼び出せるようにする。
      - 押したらどうなるか分かるメッセージをtitle属性でツールチップで表示する。
  - 4段目:
    - 6. 「URL取得」ボタン:ラベル「URL」
      - 宙の辻のURLにクエリストリングを渡して、開くURLを取得する。
        - 現在と同じ画面が開けるように、位置情報、日時情報、表示天体をURLのクエリストリングに設定して、URLをクリップボードのコピーする。
        - 日時固定の場合のURLと、URLにアクセスした時のアクセス日時を開く場合のURLを、ダイアログ画面で、選択できるようにする。
            - 位置情報の日時を確認する「日時固定(YYYY年MM月DD日hh:mm)」ラベルと「アクセス日時」ラベルの2つと、それぞれにコピーボタンがある。
            - URLをクリップボードにコピーすると、次の確認メッセージ「現在の状態で宙の辻を開くURLをクリップボードにコピーしました。」（window.alertで、OK）が表示される。
- 2列目:
  - 1段目/3段目:
    - 7. ラジオボタン
      - ユーザーが画面をタップした時に、観測点を入力させるか、目的点を入力させるか、切り替えるためのもの。
- 3列目:
  - 1段目:
    - 8. 「観測点:」テキストボックス:
      - 地名、およその住所、緯度/経度から、地図上に観測点を設定する。
      - プレースホルダー:「地名 住所 緯度,経度」
  - 2段目:
    - 9. 「観測点標高:」テキストボックス:
      - 観測点の緯度/経度から標高を設定する。
      - 背景:灰色
      - 制約:読み取り専用
  - 3段目:
    - 10. 「観測点高:」テキストボックス:
      - 観測点の高さを設定する。
      - プレースホルダー:「追加高さ m」
  - 4段目:
    - 11. 「目的点:」テキストボックス:
      - 地名、およその住所、緯度/経度から、地図上に目的点を設定する。
      - プレースホルダー:「地名 住所 緯度,経度」
  - 5段目:
    - 12. 「目的点標高:」テキストボックス:
      - 目的点の緯度/経度から標高を設定する。
      - 背景:灰色
      - 制約:読み取り専用
  - 6段目:
    - 13. 「目的点高:」テキストボックス:
      - 目的点の高さを設定する。
      - プレースホルダー:「追加高さ m」
- 各列共通
  - 7段目:均等に横幅いっぱいで配置
    - 14. 「辻ライン」トグルボタン:
      - 辻ラインを表示するためのボタン。
      - 押されている状態の時は、ラベル名を太字で、ボタン色を黄色にする。
      - 初期状態:押されている状態
    - 15. 「標高グラフ」トグルボタン:
      - 標高グラフを表示するためのボタン。
      - 押されている状態の時は、ラベル名を太字で、ボタン色を黄色にする。
### 仕様詳細:
- 14. 「辻ライン」トグルボタン:
  - 観測点標高、目的点標高、観測点視高度を利用して、垂直に立てた棒にできる影の長さの計算を応用して、観測点から目的点の方向に天体が重なる位置と時刻を求めて、地図上に図示する。
    - 破線(- - -):当日の辻ライン。5分刻みの●マーカーと時刻を表示。
    - 一点鎖線(-·-·-):天体の視半径（見かけの大きさ）を考慮したエッジライン。天体の縁が目的点に接する位置を示します。
    - 点線(· · ·):前日・翌日の辻ライン。
- 15. 「標高グラフ」トグルボタン:
  - 観測点から目的点までの直線を100mおきに標高を取得してグラフ表示する。
    - 画面下4分の1に表示する。
    - 国土地理院の標高タイルを利用して、キャッシュを利用できるような形で取得する。しかし、ローカルストレージやappStateには、キャッシュは保存しない。
    - 国土地理院の標高タイル取得は、アルゴリズムで並列化しておく。実運用で、並列度を変更して運用できるようにする。
- 共通機能/デザイン
  - テキストボックス:
    - 未入力でフォーカスが外れた場合、元の値を復元する。
    - title属性でテキストボックスの内容をツールチップで表示する。
    - input:type=textは、最大文字数150文字とする。
    - XXS対策で、入力内容をタグエスケープする。
    - フォーム、appState、ローカルストレージで、整合性が取れるようにする。
  - 背景色:
    - 1〜7段目は、背景白で、少し透明で、背景の地図が見える。
