# デッサン（dessin）
-------------------------------------------------------------------------------
-------------------------------------------------------------------------------
## 辻検索メニュー
-------------------------------------------------------------------------------
-------------------------------------------------------------------------------


### 目的:
- 例えば、富士山の右斜め上に月がある日付と時間を求められるようにするため。
- 辻ラインは、目的点緯度経度、観測点標高、目的点標高、観測点視高度から位置を逆算するため、精度が出ないので、高い精度のものが必要なため。
### 概要:
- 機能概要
  - 辻検索
    - 位置情報から、オフセットありで、リストアップ対象の許容範囲ありで、検索中心からどれくらい離れているかを、検索期間内で日付と時刻のリストアップをする。
    - 算出対象の天体は、天体リストで表示がオンになっているものを対象とする。
- デザイン概要
  - 0段目:
    - 1. 「辻検索」ラベル:
      - 太字、左寄せ
    - 2. 「辻検索」トグルボタン:
      - 中央寄せ
      - 押されている状態の時は、ラベル名を太字で、ボタン色を黄色にする。
    - 3. 「▼」ラベル:
      - 右寄せ
    - 共通事項:
      - タップすると、辻検索メニューが開閉する。
      - 背景グレー、不透明
      - 初期状態:閉じている。
  - 1段目:
    - 4. 「検索期間:」テキストボックス:
      - 検索期間を日で指定する。
      - 初期値:365
      - 制約:1〜1461(365+365+365+366、4年間)、ステップ1
      - プレースホルダー:「最大1461(4年間)」
  - 2段目:
    - 5. 「基準方位角:」テキストボックス:
      - 基準方位角を観測点緯度経度と目的点緯度経度から算出
      - 初期値:算出値
      - 制約:0〜360、ステップ0.01
      - プレースホルダー:「北から時計回り(単位:°)」
  - 3段目:
    - 6. 「基準視高度:」テキストボックス:
      - 基準視高度を観測点標高と目的点標高から算出
      - 初期値:算出値
      - 制約:−360〜360、ステップ0.01
      - プレースホルダー:「水平から上向き(単位:°)」
  - 4段目:
    - 7. 「オフセット方位角:」テキストボックス:
      - 基準方位角からのズレ量のオフセット方位角で入力する。
      - 初期値:0
      - 制約:-360〜360、ステップ0.01
      - プレースホルダー:「基準から時計回り(単位:°)」
  - 5段目:
    - 8. 「オフセット視高度:」テキストボックス:
      - 基準視高度からのズレ量のオフセット視高度で入力する。
      - 初期値:0
      - 制約:−360〜360、ステップ0.01
      - プレースホルダー:「基準から上向き(単位:°)」
  - 6段目:
    - 9. 「オフセット方位距離:」テキストボックス:
      - 観測点でのオフセット方位角を目的点での距離に直したMath.tan()で表示する。
      - 制約:読み取り専用
  - 7段目:
    - 10. 「オフセット視高距離:」テキストボックス:
      - 観測点でのオフセット視高度を目的点での距離に直したMath.tan()で表示する。
      - 制約:読み取り専用
  - 8段目:
    - 11. 「許容範囲方位角:±」テキストボックス:
      - 検索中心から検索結果を表示する方位角の範囲
      - 初期値:15
      - 制約:0〜360、ステップ0.01
      - プレースホルダー:「検索中心から時計回り(単位:°)」
  - 9段目:
    - 12. 「許容範囲視高度:±」テキストボックス:
      - 検索中心から検索結果を表示する視高度の範囲
      - 初期値:2.5
      - 制約:0〜360、ステップ0.01
      - プレースホルダー:「検索中心から上向き(単位:°)」
  - 10段目:
    - 13. 「URL取得」ボタン:
      - 宙の辻のURLにクエリストリングを渡して、開くURLを取得する。
        - 現在の情報をURLのクエリストリングに設定して辻検索をするURLを取得できるようにする。
          - ①日時固定(年月日、時間)の場合のURLと、②日時半固定(年を「0000」とした時に、アクセスした時の年に変換し、月日と時間は、URLを取得する時の日時情報の設定に従う)と、③URLにアクセスした時の現在日時のアクセス日時を開く場合のURLを、観測点/目的点の地名検索ダイアログ画面のように、ポップアップで選択できるようにする。
            - ①位置情報の日時を確認する「日時固定(YYYY年MM月DD日hh:mm)」ラベルと、②「日時半固定(アクセス年のMM月DD日hh:mm)」ラベルと、③「アクセス日時」ラベル、の3つと、それぞれにコピーできるボタンがある。
            - URLをクリップボードにコピーすると、次の確認メッセージ「現在の辻検索を開くURLをクリップボードにコピーしました。」（window.alertで、OK）が表示される。
### 仕様詳細:
- 2. 「辻検索」トグルボタン:
  - 機能概要
    - 位置情報から、オフセットありで、リストアップ対象の許容範囲ありで、検索中心からどれくらい離れているかの精度を表示して、検索期間内で日付と時刻のリストアップをする。
    - 算出対象の天体は、天体リストで表示がオンになっているものを対象とする。
    - リストは項目別にソート可能。
    - リストの表示項目:天体ID、天体名、精度記号、精度角距離、日付、時刻、方位角、視高度、視半径、月齢、月齢アイコン
  - 精度記号
    - ◎: 天体の中心がほぼ目的点に重なる日（誤差±0.125°\[誤差範囲0.25°、およそ視半径以内\]）
    - ○: 天体の縁が目的点に接する範囲の日（誤差±0.25°\[誤差範囲0.5°、およそ視直径以内\]）
    - △: 目的点との誤差が天体の直径4倍以内の日（誤差±1°\[誤差範囲2°、およそ視直径×4以内\]）
    - -: 目的点との誤差が上記以上で、指定した許容範囲以内に収まる日
  - アルゴリズム概要:
    - 1. 観測点緯度経度と目的点緯度経度から基準方位角を算出
    - 2. 観測点標高と目的点標高から基準視高度を算出
    - 3. 基準方位角と基準視高度に、オフセット方位角とオフセット視高度を加算したものを検索中心とする。
    - 4. 日時情報メニューの日付を基準日付とする。
    - 5. 天体の数だけ繰り返す
      - 1. 基準日付から1日単位で検索期間だけ繰り返す。
        - 1. 時刻0:00から1分単位で24時間繰り返す
          - 1. 天体の方位角と視高度を算出
          - 2. 検索中心からどれくらい離れているか精度を算出して、許容範囲以内のをリストに追加する。
    - 6. 画面下部にリスト表示する。
- 共通機能/デザイン
  - テキストボックス:
    - 未入力でフォーカスが外れた場合、元の値を復元する。
    - title属性でテキストボックスの内容をツールチップで表示する。
    - input:type=textは、最大文字数150文字とする。
    - XXS対策で、入力内容をタグエスケープする。
    - フォーム、appState、ローカルストレージで、整合性が取れるようにする。
  - 背景色:
    - 1〜9段目は、背景白で、少し透明で、背景の地図が見える。
