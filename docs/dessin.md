# デッサン（Dessin）

作りたい機能の構想・設計を描くファイルです。
絵を描くように、まず全体像をラフに、そして徐々に詳細を描いていきます。

## デッサンの書き方ガイド

### 詳細レベルについて

デッサンは、以下の2つのレベルがあります。どちらでも構いません。

| レベル | 説明 | 例 |
|---|---|---|
| **ラフ（VibeCoding的）** | 会話調で「こういうものが作りたい」を伝える。細かい仕様は省略してOK | 下記「VibeCoding的な伝え方」参照 |
| **詳細（仕様書的）** | コントロールごとに番号を振り、機能・制約・確認メッセージまで書く | 下記「テキストベースの仕様書的な伝え方」参照 |

### 運用ルール

- **書ける範囲で詳しく、時間がなければラフでもOK**
- 詳細に書くほど、AIが一発で正しく実装できる確率が上がる
- ラフでも、作りたいものの目的と概要が伝われば十分
- iPhoneのメモやMacのメモなど、手元のツールで書いたものをそのまま貼り付けてOK
- 1つのデッサンが大きい場合は、小さく分割して段階的に伝える

Claudさん、こんばんは。
作りたい機能があります。
今は、Homボタンで、観測点を1つしか登録できませんが、表示天体メニューのように、縦長にリストを作成して、My観測点リストを作成したいのです。
そして、My観測点リストに登録や削除機能を追加したいのです。

メニューは、表示天体のように縦長で、日時情報の下で、My天体の上に、My観測点リスト（ラベル名は「My観測点」）をタップすると開閉できるようにして、初期値は、閉じている状態で、作成します。

My観測点リストは、1段目は、辻ライン/辻検索/標高グラフの横3列のところが横2列になっている形で、「呼び出し」ボタンと「全て登録」ボタンからなりなす。
2段目は、1列目が、その行の観測点を選択中であることを表すラジオボタンで、2列目が、項番という2桁の行番号（例：「01:」）で、3列目が、観測点のラベル名を入力する「ラベル名」テキストボックス（プレースホルダーは「ラベル名」）で、4列目が、観測点の緯度経度を入力する「緯度経度」テキストボックス（プレースホルダーは「緯度,経度」）で、1列目のラジオボタンのラベルが、2列目の項番になっていて、項番をクリックしてもラジオボタンが選択されるようにして、3列目と4列目のテキストボックスは、同じ幅にしてください。
3段目は、1段目のように、辻ライン/辻検索/標高グラフの横3列のところが横2列になっている形で、「行追加」ボタンと「行削除」ボタンからなりなす。
行追加ボタンも、行削除ボタンも、押下すると、小さな「×（閉じる）」ボタンのあるウィンドが開いて、「本当に実行しますか」というメッセージと「はい」「いいえ」ボタンがあって、２段階の実行段階を踏むようにします。

呼び出しボタンは、選択されているラジオボタンの行の緯度経度テキストボックスの内容を位置情報メニューの観測点テキストボックスに入力します。
全て登録ボタンは、全ての行の、項番とラベル名テキストボックスと緯度経度テキストボックスの組みを、ローカルストレージに保存します。この時、ローカルストレージのMy観測点リストは、行数が過不足ないように不要なものは削除します。もしかしたら、全て登録前に、まとめて、ローカルストレージのMy観測点リストを全削除してから、全登録しても、問題ないと思います。
行追加ボタンは、２段階を踏んで実行されると、ラジオボタンで選択されている行の一つ下に2段目と同じ行で、内容がまだ未入力の行を追加し、ローカルストレージにもその項番の空データを追加します。しかし、項番の最大値が99番まである時は、100番目を追加させないために、行追加ボタンは、非活性になります。
行削除ボタンは、２段階を踏んで実行されると、ラジオボタンで選択されている行を削除して、ローカルストレージのその項番のデータを削除します。しかし、項番の最大値が1番の時は、1番目を削除させないために、行削除ボタンは、非活性になります。

初期状態は、ローカルストレージの状態によって、決まります。
ローカルストレージにMy観測点リストがあるならば、それらを2段目以降に行を追加して、全て読み込みます。
ローカルスロレージにMy観測点リストがない場合は、未入力の項番1番の行がある状態です。

何か疑問点や提案はありますでしょうか。

Claudeさん、おはようございます。
My観測点リストを作り直します。

基本は、以前と同じような感じですが、詳細に作り込んだので、もう一度説明します。

目的:
My観測点リストは、今までHomボタンで、観測点を1つしか登録できませんでしたが、これを複数件登録できるようにして、簡単に呼び出せるようにするものです。

概要:
見た目の概要は、現在の表示天体メニューのように、観測点のリストが縦長に構成されて、「地図に反映」ボタンや「観測点取得」ボタン、「全て登録」ボタン、複数件の観測点のリスト、「行追加」ボタン、「行削除」ボタンがあります。

仕様詳細:
●メニューの位置
・基本的なデザインは、他のメニューのデザインを継承します。
・薄明メニューの下でMy天体の上に、My観測点リスト（ラベル名は「My観測点」）を配置します。
・タップすると開閉できるようにして、初期状態は閉じている状態です。

●メニューの内容
○コントロールの配置・デザイン
・基本的なデザインは、他のメニューのデザインを継承します。
・配置の凡例は、0️⃣:段抜き、1️⃣:1行目、2️⃣:2行目、を表します。
1段目:
・1️⃣1.「地図に反映」ボタン、1️⃣2.「観測点取得」ボタン、1️⃣3.「全て登録」ボタン。
・1.〜3.のボタンは、横幅いっぱいで、1つのボタンは横幅３分１の幅。
・どのコントロールの文字サイズも16px。
2段目〜N段目:
・1段で1件の観測点を表し、観測点のリストを表示する。
・観測点の数だけ段数が増える。
・0️⃣4.ラジオボタン、0️⃣5.観測点ID「（数字）」ラベル、1️⃣6.観測点名テキストボックス（プレースホルダー「観測点名」）、2️⃣7.観測点緯度経度テキストボックス（プレースホルダー「緯度,経度」）、2️⃣8.観測点高テキストボックス（プレースホルダー「標高 m」）。

○コントロールの機能

・制約:観測点を一意に定める数字。My観測点のキー。
・制約:最小値1から始まって、最大値1000まで。

8.観測点高テキストボックス:

・このボタンを押下すると、実行する前に、確認メッセージ「観測点リストの末尾に観測点の行を追加しますか？」（window.confirmで、OK、キャンセル）が表示される。

・制約:5.観測点IDの制約の最小値を超えて削除しようとすると、このボタンが非活性（押せない状態）になる。
・このボタンを押下すると、実行する前に、確認メッセージ「観測点リストの観測点:ID（数字）を削除しますか？」（window.confirmで、OK、キャンセル）が表示される。

      - 制約:読み取り専用

    - 9. 「観測点高:」テキストボックス:
      - 観測点の高さを設定する。
      - プレースホルダー:「追加高さ m」
  - 4段目:
    - 10. 「目的点:」テキストボックス:
      - 地名、およその住所、緯度/経度から、地図上に目的点を設定する。

## 辻検索メニュー
-------------------------------------------------------------------------------
-------------------------------------------------------------------------------


### 目的:
- 例えば、富士山の右斜め上に月がある日付と時間を求められるようにするため。
- 辻ラインは、目的点緯度経度、観測点標高、目的点標高、観測点視高度から位置を逆算するため、精度が出ないので、高い精度のものが必要なため。
### 概要:
- 機能概要
  - 辻検索
    - 位置情報から、オフセットありで、リストアップ対象の許容範囲ありで、検索中心からどれくらい離れているかを、検索期間内で日付と時刻のリストアップをする。
    - 算出対象の天体は、天体リストで表示がオンになっているものを対象とする。
- デザイン概要
  - 0段目:
    - 1. 「辻検索」ラベル:
      - 太字、左寄せ
    - 2. 「辻検索」トグルボタン:
      - 中央寄せ
      - 押されている状態の時は、ラベル名を太字で、ボタン色を黄色にする。
    - 3. 「▼」ラベル:
      - 右寄せ
    - 共通事項:
      - タップすると、辻検索メニューが開閉する。
      - 背景グレー、不透明
      - 初期状態:閉じている。
  - 1段目:
    - 4. 「検索期間:」テキストボックス:
      - 検索期間を日で指定する。
      - 初期値:365
      - 制約:1〜1461(365+365+365+366、4年間)、ステップ1
      - プレースホルダー:「最大1461(4年間)」
  - 2段目:
    - 5. 「基準方位角:」テキストボックス:
      - 基準方位角を観測点緯度経度と目的点緯度経度から算出
      - 初期値:算出値
      - 制約:0〜360、ステップ0.01
      - プレースホルダー:「北から時計回り(単位:°)」
  - 3段目:
    - 6. 「基準視高度:」テキストボックス:
      - 基準視高度を観測点標高と目的点標高から算出
      - 初期値:算出値
      - 制約:−360〜360、ステップ0.01
      - プレースホルダー:「水平から上向き(単位:°)」
  - 4段目:
    - 7. 「オフセット方位角:」テキストボックス:
      - 基準方位角からのズレ量のオフセット方位角で入力する。
      - 初期値:0
      - 制約:-360〜360、ステップ0.01
      - プレースホルダー:「基準から時計回り(単位:°)」
  - 5段目:
    - 8. 「オフセット視高度:」テキストボックス:
      - 基準視高度からのズレ量のオフセット視高度で入力する。
      - 初期値:0
      - 制約:−360〜360、ステップ0.01
      - プレースホルダー:「基準から上向き(単位:°)」
  - 6段目:
    - 9. 「オフセット方位距離:」テキストボックス:
      - 観測点でのオフセット方位角を目的点での距離に直したMath.tan()で表示する。
      - 制約:読み取り専用
  - 7段目:
    - 10. 「オフセット視高距離:」テキストボックス:
      - 観測点でのオフセット視高度を目的点での距離に直したMath.tan()で表示する。
      - 制約:読み取り専用
  - 8段目:
    - 11. 「許容範囲方位角:±」テキストボックス:
      - 検索中心から検索結果を表示する方位角の範囲
      - 初期値:15
      - 制約:0〜360、ステップ0.01
      - プレースホルダー:「検索中心から時計回り(単位:°)」
  - 9段目:
    - 12. 「許容範囲視高度:±」テキストボックス:
      - 検索中心から検索結果を表示する視高度の範囲
      - 初期値:2.5
      - 制約:0〜360、ステップ0.01
      - プレースホルダー:「検索中心から上向き(単位:°)」
### 仕様詳細:
- 2. 「辻検索」トグルボタン:
  - 機能概要
    - 位置情報から、オフセットありで、リストアップ対象の許容範囲ありで、検索中心からどれくらい離れているかの精度を表示して、検索期間内で日付と時刻のリストアップをする。
    - 算出対象の天体は、天体リストで表示がオンになっているものを対象とする。
    - リストは項目別にソート可能。
    - リストの表示項目:天体ID、天体名、精度記号、精度角距離、日付、時刻、方位角、視高度、視半径、月齢、月齢アイコン
  - 精度記号
    - ◎: 天体の中心がほぼ目的点に重なる日（誤差±0.125°\[誤差範囲0.25°、およそ視半径以内\]）
    - ○: 天体の縁が目的点に接する範囲の日（誤差±0.25°\[誤差範囲0.5°、およそ視直径以内\]）
    - △: 目的点との誤差が天体の直径4倍以内の日（誤差±1°\[誤差範囲2°、およそ視直径×4以内\]）
    - -: 目的点との誤差が上記以上で、指定した許容範囲以内に収まる日
  - アルゴリズム概要:
    - 1. 観測点緯度経度と目的点緯度経度から基準方位角を算出
    - 2. 観測点標高と目的点標高から基準視高度を算出
    - 3. 基準方位角と基準視高度に、オフセット方位角とオフセット視高度を加算したものを検索中心とする。
    - 4. 日時情報メニューの日付を基準日付とする。
    - 5. 天体の数だけ繰り返す
      - 1. 基準日付から1日単位で検索期間だけ繰り返す。
        - 1. 時刻0:00から1分単位で24時間繰り返す
          - 1. 天体の方位角と視高度を算出
          - 2. 検索中心からどれくらい離れているか精度を算出して、許容範囲以内のをリストに追加する。
    - 6. 画面下部にリスト表示する。
- 共通機能/デザイン
  - テキストボックス:
    - 未入力でフォーカスが外れた場合、元の値を復元する。
    - title属性でテキストボックスの内容をツールチップで表示する。
    - input:type=textは、最大文字数150文字とする。
    - XXS対策で、入力内容をタグエスケープする。
    - フォーム、appState、ローカルストレージで、整合性が取れるようにする。
  - 背景色:
    - 1〜9段目は、背景白で、少し透明で、背景の地図が見える。

    - 共通事項:
